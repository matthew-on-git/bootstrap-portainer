## Intent

**Problem:** There is no idempotent, re-runnable installation script for deploying Portainer CE via Docker Compose on Ubuntu servers, following the established patterns from bootstrap-supabase.

**Approach:** Create an `install.sh` script that deploys Portainer CE using Docker Compose with pinned image versions, interactive configuration prompts, and support for both interactive and unattended (-y flag) modes.

## Boundaries & Constraints

**Always:**
- Follow bootstrap-supabase patterns (idempotent, re-runnable, pinned versions)
- Use Docker Compose (not docker run) for deployment
- Support Ubuntu 22.04/24.04
- Use `portainer/portainer-ce:sts` image (pinned version)
- Expose ports 9443 (HTTPS UI) and 8000 (Edge tunnel)
- Mount Docker socket and persistent volume
- Save configuration to `.install.conf` for re-runs
- Use shared logging library (log_info, log_warn, log_error, die from lib/log.sh)

**Ask First:**
- Changes to default ports (9443, 8000)
- Adding TLS/HTTPS configuration
- Adding SMTP or authentication settings
- Supporting additional Portainer Agent deployments

**Never:**
- Use `:latest` tags for images
- Break idempotency (must be safe to re-run)
- Install Docker (assume it's already installed)
- Support non-Ubuntu distributions
- Add Portainer Business Edition features

## I/O & Edge-Case Matrix

| Scenario | Input / State | Expected Output / Behavior | Error Handling |
|----------|--------------|---------------------------|----------------|
| Fresh install | First run, no existing config | Creates `.install.conf`, deploys Portainer via docker compose, container running | Die if Docker unavailable |
| Re-run existing | Config exists, container running | No-op (already running), reports status | Warn if config corrupted |
| Unattended mode | `-y` flag passed | Uses all defaults, no prompts | Die on missing required values |
| Custom port | User specifies non-default port | Uses specified port in compose file and container | Validate port is available |
| Docker unavailable | Docker not installed/running | Script fails with clear error | Die with message about Docker requirement |

## Design Notes

**Compose File Template:**
```yaml
services:
  portainer:
    container_name: portainer
    image: portainer/portainer-ce:sts
    restart: always
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - portainer_data:/data
    ports:
      - 9443:9443
      - 8000:8000

volumes:
  portainer_data:
    name: portainer_data

networks:
  default:
    name: portainer_network
```

**Key Differences from bootstrap-supabase:**
- Single container (not multi-service stack)
- No database to configure
- No SMTP/auth complexity
- Simpler overall (good first project pattern)
